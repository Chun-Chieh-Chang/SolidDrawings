1. 產品定位

3D-Builder 是以 OpenCASCADE 為幾何核心、受 SolidWorks 操作流程啟發的輕量化參數式 3D CAD 工具，聚焦零件建模、基礎工程圖、標準格式交換與教育/中小型設計場景。

1.0 前禁止宣稱：
  • 完整取代 SolidWorks。
  • 原生支援 .sldprt / .sldasm。
  • 支援大型工業裝配。
  • 支援完整鈑金、模具、Simulation、PDM。

初期主打價值

  • 輕量、跨平台、開放格式。
  • 現代 Web/Electron 桌面體驗。
  • 基於 OpenCASCADE 的真 B-Rep 幾何核心。
  • 參數化零件建模與 STEP / STL / IGES 交換。

4. Phase 0 到 Phase 6

Phase 0：產品化基礎整理

  • 改用 .3dbpart，停止將 JSON 偽裝為 .sldprt。
  • 建立 com.3dbuilder.part 版本化 schema。
  • 建立規格文件與測試基礎。
  • 移除 legacy sketchPoints，統一 graph sketch model。

Phase 1：Alpha

穩定完成「建立草圖 → 加約束 → Extrude / Cut → Fillet / Chamfer → 匯出 STEP/STL」。

Phase 2：Private Beta

標準測試模型、Topological Naming 初版、Rollback 完整化、STEP 匯入與 UX 打磨。

Phase 3：Public Beta

安裝包、後端程序管理、crash reporting、文件與使用者回饋。

Phase 4：1.0

可正式交付的輕量參數化零件 CAD。

Phase 5：1.5

工程圖、匯入匯出、材料與屬性強化。

Phase 6：2.0

裝配體、mate solver、干涉檢查、BOM 與進階特徵。

6. 測試、效能與 Release Gate

1.0 效能目標

項目目標
啟動時間小於 5 秒
開啟簡單零件小於 2 秒
Rebuild 20 features小於 1 秒
草圖拖曳回饋60 FPS 目標
STEP export 小零件小於 3 秒
PDF drawing export小於 3 秒

1.0 Release Gate

  • 至少 10 個標準測試零件。
  • 可儲存/開啟 .3dbpart。
  • 可匯出 STEP/STL/IGES。
  • 可完成基本三視圖 PDF。
  • TypeScript、backend pytest、E2E 建模流程通過。

7. PDCA 執行要求

每次開發或修訂完成後必須執行：
npm run pdca:check
  1. Plan：讀取本文件與 task_plan.md。
  2. Do：完成最小可驗證變更。
  3. Check：執行 npm run pdca:check 與任務對應測試。
  4. Act：若不符合 Plan，在 DEV_LOG.md 使用 RCA/CAPA 模板記錄並修正。
