# Concept: PDCA Evolution (自進化循環)

## 1. 核心定義 (Definition)
PDCA 循環（Plan-Do-Check-Act）是本系統實現「自進化」的核心引擎。它不只是一個開發流程，更是一個「智慧過濾器」，確保每一次的變更都能沉澱為持久的規則。

## 2. 循環四階段 (The 4 Phases)

### [Plan] 診斷與策略
- **第一性原則**：從原始需求出發，不盲從現有路徑。
- **風險掃描**：執行「依賴掃描」與「脆弱點分析」。

### [Do] 精準執行
- **外科手術式修改**：追求最小變動量，避免副作用。
- **免疫紀錄**：同步更新 `DEV_LOG.md` 中的 RCA/CAPA，建立系統的「免疫記憶」。

### [Check] 水平展開確效
- **水平展開 (Horizontal Expansion)**：修復 A 點的 Bug 後，自動檢核全專案中是否存在 B、C、D 等同類風險。
- **運行驗證**：npm run build 與 Console 零錯誤標準。

### [Act] 知識昇華
- **回饋全域**：將修復過程中發現的通用規律，寫回 `SkillsBuilder` 的全域規則庫中。
- **自進化完成**：下一次啟動時，系統將具備全新的防禦與開發能力。

## 3. 系統自進化哲學
「今日的 Bug 是明日規則的養料。」
循環絕不終止，每一輪循環的結束，都是系統能力邊界的一次擴張。
